# ESM/CJS execution-semantics conformance suite

Location: `conformance/`. This suite is separate from the oracle
(`oracle/`): the oracle checks agreement between Diffpack and Rolldown on
hand-specified expectations; the conformance suite measures all three
bundlers (Diffpack, Rolldown, esbuild) against **Node's own unbundled
execution semantics**.

Headline result (2026-07-21, Node v22.21.0, this machine): **Diffpack
silently produces wrong output on 8 of 48 fixtures.** Five of those are
Diffpack-specific (both reference bundlers get them right or refuse); three
are deviations shared by all three bundlers. Diffpack additionally emits
five bundles that Node cannot parse at all (top-level `await` /
`import.meta` inside its CommonJS output) — the build "succeeds" but the
program is invalid, which contradicts the project's honest-hard-error
policy. Diffpack produced **zero** honest build errors across the suite.

> **Update (same day):** `order-side-effect-imports` — the worst finding —
> is FIXED (two root causes: `collect_dependencies` alphabetized import
> specifiers, and `static_execution_order` seeded its walk from every
> module in id order instead of from the entry). The fix also corrected a
> live CSS-cascade inversion on the `integration/vite-react-reference`
> fixture (`App.css` overrides lost to `index.css`). Pinned by
> `side_effect_imports_execute_in_import_order_not_id_order` and
> `extracted_css_follows_import_order_so_the_cascade_ties_break_correctly`
> in `src/bundler.rs`; the suite now reports diffpack 35 pass / 7
> wrong-output. The matrix below is the original 2026-07-21 run kept for
> the record; `conformance/results.json` holds the current run.

## Methodology

- ~48 minimal fixtures under `conformance/fixtures/<name>/`, one semantic
  point each. Every fixture's `expected.txt` is the stdout of running the
  **unbundled** entry directly under Node (`node run.mjs --update-expected`
  regenerates them by actual execution; nothing is hand-written). A nonzero
  expected exit is recorded in `expected-exit.txt`.
- Exception (1 fixture): `mixed-cycle-esm-cjs-eager` cannot run unbundled —
  Node refuses eager `require(esm)` inside a cycle with
  `ERR_REQUIRE_CYCLE_MODULE` (a loader admission policy, not observable
  execution semantics). Its ground truth is the *agreement* of Rolldown and
  esbuild, generated by actually building and running both and requiring
  identical stdout and exit class. This is documented in that fixture's
  `node-cannot-run.md`; a PASS there means "matches the consensus bundler
  linearization", not "matches Node".
- Bundler invocations:
  - **Diffpack**: `target/release/diffpack bundle <entry> <out>` (built from
    this repo). The output is executed as `.cjs`. This is not a harness
    choice imposed on Diffpack: Diffpack has no output-format flag, its own
    oracle executes output as `.cjs`, and the emitted bundle literally
    begins with `module.exports=(()=>{`, i.e. it *is* CommonJS. (Renaming
    the output `.mjs` was probed: it rescues the `import.meta`/TLA fixtures
    but breaks nearly everything else with `ReferenceError: module is not
    defined`, so no extension makes the current emit coherent for those
    features.)
  - **Rolldown 1.2.0** (pinned, same as the oracle): `format: "esm"`,
    `codeSplitting: false`, `treeshake: false`, output executed as `.mjs`.
  - **esbuild 0.28.1** (pinned): `bundle: true`, `format: "esm"`,
    `platform: "node"`, output executed as `.mjs`.
- Each bundle is executed by the same Node (v22.21.0); stdout and exit
  class are compared to the ground truth. stderr is *not* compared (error
  message text legitimately differs between implementations).
- Classification per bundler per fixture:
  - **PASS** — stdout matches and exit class (zero/nonzero) matches.
  - **WRONG-OUTPUT** — the bundle ran but printed something different.
    This is the worst class: silent semantic corruption.
  - **BUILD-ERROR** — the bundler refused. For Diffpack this is its
    documented honest-hard-error policy; recorded as a gap, never hidden.
  - **RUNTIME-ERROR** — the bundle crashed where the unbundled program
    succeeded (includes "build succeeded but emitted invalid JS").
- The runner (`conformance/run.mjs`) prints the matrix, writes
  `conformance/results.json`, and exits nonzero iff Diffpack has any
  WRONG-OUTPUT. Reproduce with:

  ```console
  cargo build --release
  cd conformance
  npm ci
  node run.mjs
  ```

## Diffpack gap list

### A. Silent wrong output — these are bugs

Diffpack-specific (at least one reference bundler matches Node):

1. **order-side-effect-imports** — the worst finding. Entry does
   `import "./b.js"; import "./a.js"; import "./c.js"`. Node (and both
   references) execute `b, a, c`; Diffpack's concatenated output executes
   `a, b, c` — side-effect module evaluation order is not the import order.
   The oracle's own `side-effect-order` fixture never caught this because
   its import order happens to coincide with the order Diffpack picks.
2. **esm-namespace-import** — namespace object keys are in source
   declaration order (`beta,alpha`) instead of the spec-mandated sorted
   order (`alpha,beta`). Same root cause family as the previous item
   (export enumeration order).
3. **esm-this-undefined** — top-level `this` in an ESM entry must be
   `undefined`; Diffpack's CJS-wrapped output leaks `this === exports`
   (`[object Object]`). Both references rewrite it correctly.
4. **cjs-esmodule-marker** — importing default from a real `.cjs` file that
   sets `exports.__esModule = true; exports.default = ...`: Node gives the
   whole `module.exports` object as default (Node ignores `__esModule`);
   Diffpack applies bundler-style `__esModule` interop and gives
   `exports.default`. Notably both Rolldown and esbuild match Node here for
   a genuine `.cjs` file, so Diffpack deviates from Node *and* from both
   references.
5. **namespace-frozen** — `Object.isFrozen(ns)` must be `false` for a
   module namespace exotic object (its properties report `writable: true`
   even though writes throw); Diffpack's sealed namespace reports `true`.
   Everything else about the fixture (TypeError on write/add,
   `Symbol.toStringTag`) matches. Minor, but observable.

Shared deviations (all three bundlers produce the same wrong output — lower
priority for Diffpack, but still deviations from Node):

6. **esm-default-fn-name** — `export default function () {}` must produce a
   function named `"default"`; Diffpack names it `"__diffpack_default_0"`
   (Rolldown/esbuild: `"anon_default"`).
7. **esm-reexport-star-from-cjs** — namespace keys of
   `export * from "./legacy.cjs"` come out in insertion order (`foo,bar`)
   instead of sorted (`bar,foo`). Values are correct.
8. **dynamic-import-cjs** — `import("./legacy.cjs")` where
   `module.exports = { tag }`: Node's namespace exposes only `default`
   (`"tag" in ns` is false); all three bundlers also expose `tag` as a
   named property.

### B. Build "succeeds" but the emitted bundle is invalid JS

These violate the honest-hard-error policy: Diffpack reports a successful
build, then Node rejects or mis-executes the output. They surface as
RUNTIME-ERROR in the matrix but the right fix is either real support or a
hard build error.

- **tla-basic**, **tla-dep-order**, **tla-dynamic-import-order**,
  **dynamic-static-same-namespace** — top-level `await` is passed through
  into the CommonJS output: `SyntaxError: await is only valid in async
  functions and the top level bodies of modules`. (ORACLE.md explicitly
  excludes TLA from the current target, so per policy these should be
  build-time refusals, not broken emits.)
- **esm-import-meta-url** — `import.meta` is passed through into the
  CommonJS output: `SyntaxError: Cannot use 'import.meta' outside a
  module`.

### C. Hard-errored unsupported features

None. Diffpack refused zero fixtures. Given category B, that is not a
compliment: the TLA/`import.meta` cases *should* be here.

### Shared limitation (all three bundlers)

- **dynamic-import-concat-literal** — `import("./mo" + "d.js")` is not
  constant-folded by any of the three bundlers; the specifier is left as a
  runtime dynamic import, which fails once the bundle is moved away from
  the sources (`ERR_MODULE_NOT_FOUND`). Unbundled Node runs it fine.

## Where the reference bundlers deviate from Node (context)

These are cases where Rolldown and/or esbuild also fail against Node's
semantics — useful calibration for what "industry standard" actually
achieves, and areas where Diffpack is *ahead*:

- **esm-tdz-class-caught** (both) — reading a class binding in TDZ across a
  cycle throws `TypeError` (from their interop wrapper) instead of
  `ReferenceError`. Diffpack passes.
- **esm-tdz-const-caught** (esbuild) — esbuild reads `undefined` instead of
  throwing; Rolldown and Diffpack both pass.
- **esm-cycle-tdz-uncaught** (esbuild) — esbuild's bundle exits 0 where
  Node's unbundled run (and Diffpack's and Rolldown's bundles) fail with an
  uncaught `ReferenceError`. TDZ emulation is simply not implemented.
- **cjs-typeof-exports** (both) — `typeof __filename` becomes `undefined`
  in their ESM-format output; Diffpack's CJS output preserves it. (This one
  is partly an artifact of output format choice.)
- **namespace-frozen** (both) — Rolldown and esbuild *refuse to build* a
  valid program (`ns.a = 99` where `ns` is a namespace import is legal JS
  that throws `TypeError` at runtime; Node runs the program). Diffpack
  builds and runs it, differing only on the `Object.isFrozen` detail.
- **esm-default-fn-name**, **esm-reexport-star-from-cjs**,
  **dynamic-import-cjs** — shared with Diffpack, see above.

## Fixture inventory

48 fixtures. Coverage: ESM named/default/namespace imports, export
renaming, re-export chains, `export *` including ambiguity elision and
local shadowing, `export *` from CJS, self-referential namespace,
`import.meta.url`, top-level `this`, string-literal export names, TDZ
(caught const, caught class, uncaught cycle), live bindings (mutable `let`,
`export { x as default }`), hoisted functions across cycles, ESM/CJS/mixed
cycles (eager and lazy), CJS default interop (object, primitive,
`__esModule` marker), CJS named/namespace imports, `require` semantics
(cache identity, conditional/lazy, `exports` vs `module.exports`
reassignment, `typeof exports`/`this === exports`), dynamic import
(ESM/CJS/computed specifier/static-dynamic namespace identity), top-level
await (basic, in-dependency, with dynamic import ordering), evaluation
order (side-effect imports, diamond executes once, import hoisting),
namespace liveness and freeze semantics, and JSON modules (ESM import
attribute and CJS `require`).

All fixtures are deterministic: no timers, no randomness, and each TLA
fixture has a single possible interleaving.
