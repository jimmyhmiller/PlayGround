Living Whiteboard — Sim ↔ Visual Strategy Explorer

The simulator is a discrete-event engine: packets jump between nodes at exact sim-time nanoseconds. The screen is wall-clock and ~60 fps. Six visual strategies bridge that gap — each turns the same event stream into animated packets differently. This page runs one shared sim through all six at once so you can see the trade-offs directly.

The two knobs that define the gap. Under the host's lockstep mapping visual_now = sim_now × 1e-9, so both clocks advance together.

multiplier — sim time vs wall time

Sim-seconds per wall-second. At ×1 a 8 ms edge is on screen for 8 ms of wall — a blink. Slowing it is the only way SimMirror becomes legible.

k — visual latency stretch

Event-history strategies set arrive = emit + sim_latency × k. Big k stretches each hop into something the eye can follow, decoupling apparent travel time from real latency.

SimMirror — low latency makes packets vanish

SimMirror animates each packet for its true latency — no k stretch. These hops are 1 ms. At real time (×1) a packet crosses an edge in 1 ms — shorter than a single 16 ms frame — so the renderer literally samples between packets and the screen is empty on almost every frame, even though dozens per second are streaming through underneath.

The sim is busy; your eye just can't catch it. Drag slow-motion down and the same traffic re-materialises — that's the only lever SimMirror gives you for legibility.

Replay — the causal-clamp runaway

Request → router → worker → reply back to the client — a cycle. Replay drops nothing, but with a big k (410) every hop is stretched to seconds, and the causal clamp makes each hop wait for the previous one's stretched arrival. Because the reply returns to G, the next request can't visually start until the previous reply has visually arrived — so Replay serializes the entire loop.

The sim fires 12 req/s and finishes each in ~53 ms. But each visual round trip takes ~11 s, so Replay falls further behind every second — watch behind by climb without bound. You end up watching ancient request #— crawl across while the sim has raced thousands ahead: you stop seeing current activity at all. Drop k below ~2 (round trip < request spacing) and the runaway collapses — Replay snaps back to the sim.

Visual cues: red + halo = an acausal packet (shown before its cause arrived), with a dashed link to where its cause actually is  ·  faint hollow = a dropped packet (you see what's thrown away)  ·  purple ring = a retroactively promoted cause.

RateSampled — effects before their causes

RateSampled has no causal clamp. Each hop is drawn starting at its own sim-time, and never waits for the hop that caused it to visibly arrive. Since every hop is stretched by k, a packet departs A→B, B→C, even C→S while the packet that feeds it is still crawling across the first edge — it spawns from a node nothing has reached yet.

Watch the red dots: each is a packet on screen before its own cause has arrived. (The clamped strategies below hold each hop until its cause lands, so the chain plays out one stage at a time — no red.)

DropOrphans — no spawning-from-idle, but it drops the most

Same throttle, but it refuses to show a packet leaving a node that has had no shown arrival — so you never catch one spawning from a totally idle stage (the worst red case above). The price is brutal: suppressing a cause cascades to its whole downstream subtree, so it drops the most by far — watch drop %. The picture is tidy but sparse, badly under-reporting how busy the system is. (It guards the node, not the individual item — it still can't promise every packet's exact cause was shown.)

CausalRate — time-travel the cause

Same throttle as DropOrphans, but instead of dropping a needed-but-throttled cause it retroactively flashes that cause back. Watch (this one runs in slow motion): packets get throttled away as faint ghosts — but when a surviving packet needs one of those dropped causes, CausalRate fires a purple ↩cause ripple at that node, resurrecting the cause after the fact so the chain stays whole.

That's the unique move: DropOrphans (above) throws the cause away forever; CausalRate brings it back just-in-time. It keeps true per-item causality and drops far less — at the cost of showing causes slightly out of order.

BundleSummarized — one dot stands for many

Coalesces everything on an edge within a 100 ms sim-window into a single packet. At high rate that means a dozen events collapse to one dot — so the chain stays legible, but the count is lost: a burst of 2 and a burst of 50 look identical. Watch events represented race ahead of shown.
