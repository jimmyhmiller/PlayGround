Live & Typed · The Lab

Hot-reload a running, statically-typed program.

A tiny language where you edit types and functions while the program runs. Consistent changes migrate live values automatically. When running code reaches something the update broke, it doesn't crash and the update isn't rejected — it pauses, you repair it, and it resumes. Every committed state is checked well-typed. This is the whole design, running in your browser.

Move 1

Update at a safe point

Edits apply between ticks — never mid-computation.

Move 2

Migrate live values

Old values convert lazily on first touch; auto-derived where possible.

Move 3

Pause on a condition

A broken function or un-migratable value raises a condition and freezes.

Move 4

Repair & resume

You supply the missing code live; the tick re-runs. No restart.

What this proves — and what it doesn't

It shows the design is coherent and runnable: one primitive — a type condition raised at the point of use, with freeze → repair → resume — reconciles hot updates, lazy typed migration, and static soundness. The ✓ well-typed badge is an enforced check, not a claim: nothing ill-typed is ever committed, because a pause rolls the in-flight tick back.

It deliberately takes the easy version of the hard corners, so don't over-read it:

Single-threaded — quarantine is trivial here. The real test is two concurrent processes sharing a value mid-migration.
Pauses happen at call-entry, before effects — so "resume" is a clean frame restart. Mid-expression pauses after a side effect are the hard case.
Types change monotonically; identity is name+version, not content-addressed.
